SECTION 4: KEY MAPPINGS REFERENCE
═══════════════════════════════════════════════════════════════════════════════════════════════════════════

PAYMENT MODE MAPPING (Column: Mode)
├─ RTGS           → RTGS
├─ AIRTEL         → AIRTEL / CASH
├─ CASH           → AIRTEL / CASH
├─ CHEQUE         → CHQ / DD
├─ DD             → CHQ / DD
└─ ONLINE_PAYMENT → ONLINE_PAYMENT


RECEIPT STATUS MAPPING (Column: Status)
├─ B  → Bounced
├─ C  → Cleared
├─ D  → Deposit
├─ NA → Pending
└─ X  → Cxn (Cancellation)


RECEIPT TYPE MAPPING (Column: Receipt Type)
├─ OD                 → OD (Interest/Charges)
├─ FC                 → Settlement (Foreclosure)
├─ Part Payment       → Part Payment
├─ Settlement         → Settlement
├─ Sale/EMD receipt   → Settlement
└─ [Fallback to RECEIPT CAT if not found]
   ├─ OD              → OD
   ├─ OTHER OD        → OTHER OD
   └─ -               → [blank]


RECEIPT SOURCE MAPPING (Column: Receipt Source)
├─ BBPS               → BBPS
├─ CCP                → CCP - QR (normalized)
├─ CCP - QR           → CCP - QR
├─ CCP - Bitly        → CCP - Bitly
├─ CCP - BITLY        → CCP - Bitly (normalized)
├─ CHOLAONE DIRECT    → CHOLAONE DIRECT
└─ [others preserved as-is]


DISABLE LIST FLAGS
├─ Ag Level cash mode: From "Agreement Level Disble" sheet
│  Values: "CIF Level > 1.95 L", "Case Level > 1.95 L", etc.
│  Meaning: This agreement NOT allowed to pay via cash ❌
│
└─ CIF LEVEL: From "CIF Level Disable" sheet
   Values: "Not Required", "To be Disabled", "CIF Level > 1.95 L"
   Meaning: This customer NOT allowed to pay via cash ❌

SECTION 5: THE MONTHLY WORKFLOW (How to Use This Every Month)
═══════════════════════════════════════════════════════════════════════════════════════════════════════════

MONTH N+1 WORKFLOW:
┌──────────────────────────────────────────────────────────────────┐
│ 1. Gather 4 Input Files                                          │
│    ├─ DCR.xlsb (this month's collections)                       │
│    ├─ To_be_Disabled.xlsb (updated compliance lists)             │
│    ├─ Employee Master.xlsx (unchanged usually)                   │
│    └─ DHC_Working_Output_From_MONTH_N.xlsx (prev month output) ◀─│
│       (← This becomes "Previous DHC Working" input)              │
│                                                                  │
│ 2. Upload to Streamlit App → Click "Process Files"              │
│    └─ Runs full ETL pipeline (40 seconds)                       │
│                                                                  │
│ 3. Review Output                                                 │
│    ├─ Preview all 5 summary sheets                              │
│    ├─ Check "New Agreements" metric                             │
│    └─ Download Excel workbook                                    │
│                                                                  │
│ 4. (IF) New Agreements Found                                     │
│    ├─ Open "Look Up Master" tab in output                       │
│    ├─ Find rows with blank CIF_NO                               │
│    ├─ Manually lookup CIF from your LMS                         │
│    ├─ Fill in CIF_NO, ZONE_NEW, SUB_REGION                      │
│    └─ Save the file                                              │
│                                                                  │
│ 5. Next Month                                                    │
│    └─ Use THIS MONTH's output as "Previous DHC Working" input   │
│       (CIF mappings carried forward → fewer manual entries)      │

SECTION 6: WHAT CHANGED FROM MANUAL FORMULAS
═══════════════════════════════════════════════════════════════════════════════════════════════════════════

OLD WAY (Manual Excel Formulas):
├─ VLOOKUP formulas pointing to external files ❌
├─ Broken links when files moved → #REF! errors ❌
├─ Copy-paste mistakes in formulas ❌
├─ Formulas slow for 100K+ rows ❌
├─ Hard to audit (formulas hidden in cells) ❌
├─ Manual COUNTIFS/SUMIFS prone to typos ❌
└─ Takes 20-30 minutes per month ❌


NEW WAY (Automation):
├─ Python code (visible, auditable) ✅
├─ No external links — self-contained ✅
├─ Deterministic logic (same inputs = same output) ✅
├─ Fast (40 seconds for 100K+ rows) ✅
├─ Easy to debug (error messages are clear) ✅
├─ Exact aggregations (PIVOT_TABLE replicas) ✅
└─ Takes 40 seconds per month ✅
